# Ankiアドオンの作り方

## 1. ディレクトリ構造とファイル構成

### アドオンの場所
- **macOS**: `~/Library/Application Support/Anki2/addons21`
- **Windows**: `C:\Users\<User>\AppData\Roaming\Anki2\addons21`

### 基本的なファイル
1.  **`__init__.py`** (必須)
    -   アドオンのエントリーポイント。
    -   このファイルが存在しないとAnkiはアドオンとして認識しない。
    -   `aqt` (GUI) や `anki` (バックエンド) モジュールをインポートして処理を記述する。

2.  **`manifest.json`**
    -   アドオンのメタデータ（パッケージ名、バージョンなど）を定義する。

3.  **`config.json`** (任意)
    -   ユーザーが変更可能な設定値を保存するJSONファイル。

4.  **`config.md`** (任意)
    -   設定項目の説明を記述するMarkdownファイル。Ankiのアドオン設定画面に表示される。

5.  **`user_files/`** (任意)
    -   アドオンのアップデート時に削除されたくないユーザー固有のデータ（音声ファイルや画像など）を保存するフォルダ。
    -   アドオンルート直下に作成する。

## 2. Pythonコードの基本

### 主要なモジュール
-   **`aqt`**: AnkiのGUI部分 (PyQt5ベース)。
    -   `mw`: メインウィンドウオブジェクト。`from aqt import mw` で取得。
    -   `mw.col`: コレクション（データベース）操作への入り口。
-   **`anki`**: Ankiのバックエンドロジック（スケジューリング、カード操作など）。

### よく使うAPI
-   **情報の表示**: `aqt.utils.showInfo("メッセージ")`
-   **カード枚数の取得**: `mw.col.cardCount()`
-   **カードの検索**: `mw.col.findCards("tag:example")` (カードIDのリストを返す)
-   **ノートの編集**:
    ```python
    note = card.note()
    note['フィールド名'] = '新しい値'
    note.flush() # 変更を保存
    ```
-   **データベースのリセット**: `mw.reset()` (変更を画面に反映させるために必要)

### フック (Hooks)
Ankiのイベントに割り込んで処理を実行する仕組み。
-   `anki.hooks.addHook("イベント名", コールバック関数)`
-   例: `setupEditorButtons` (エディタにボタン追加), `reviewCleanup` (レビュー終了時) など。

### モンキーパッチ
フックがない場合、既存の関数をラップして動作を変更する。
-   `anki.hooks.wrap(元の関数, 新しい処理, "before/after")` を使うと便利。

## 3. デバッグ方法
-   **標準エラー出力**: `sys.stderr.write("デバッグ情報\n")` でログを出力可能。
-   **Ankiコンソール**: Anki内でデバッグコンソールを起動して、変数の値を確認できる。

## 4. 配布・共有
-   AnkiWebにアップロードする際は、アドオンフォルダの中身（`__init__.py`など）を直接選択してzip圧縮する（フォルダごと圧縮しない）。
-   `.pyc`ファイルや`__pycache__`ディレクトリは含めない。
